# 2D Soft Renderer (SDL2)
一个基于SDL2实现的简易2D软渲染器。核心功能是绘制带有呼吸效果和边缘平滑过渡的彩色圆形，支持鼠标交互（跟随鼠标的动态圆形）。

---

## 功能特点
- 基于SDL2图形库实现跨平台窗口创建与渲染
- 绘制带平滑边缘过渡效果的圆形
- 动态呼吸效果（圆形半径随时间周期性变化）
- 鼠标跟随效果（第一个圆形跟随鼠标位置移动）
- 多圆形独立的动态效果配置

## 环境依赖
- C++ 11及以上编译器
- SDL2 开发库（需配置SDL2的头文件和链接库）

## 编译运行
### Windows 编译（MinGW）
```bash
# 假设SDL2已配置好环境变量，编译命令：
g++ main.cpp -o soft_renderer -lSDL2 -lSDL2main -std=c++11
# 运行程序
soft_renderer.exe
```

### Linux 编译
```bash
# 安装SDL2依赖
sudo apt-get install libsdl2-dev
# 编译
g++ main.cpp -o soft_renderer -lSDL2 -std=c++11
# 运行
./soft_renderer
```

## 核心代码说明
### 1. 基础数据结构
- `Vec2`：二维向量结构体，用于表示坐标点，重载了加减乘运算符
- `Color`：颜色结构体，包含RGBA四个通道（8位无符号整数）

### 2. SoftRenderer 渲染类
- 封装SDL2窗口、渲染器的创建与销毁
- 核心方法：
  - `clear()`：清空窗口背景
  - `draw_circle()`：绘制带平滑边缘的圆形（核心逻辑：计算像素到圆心距离，通过三次方衰减实现边缘过渡）
  - `present()`：刷新渲染结果到窗口

### 3. 主循环逻辑
- 事件监听：处理窗口关闭、鼠标移动事件
- 呼吸效果：通过正弦函数周期性修改圆形半径
- 渲染流程：清空背景 → 绘制三个动态圆形 → 刷新窗口

## 运行效果
1. 窗口尺寸：800x600
2. 三个动态圆形：
   - 第一个圆形：跟随鼠标移动，半径随呼吸效果变化（80±8）
   - 第二个圆形：固定X坐标（400），Y坐标随正弦函数波动，半径变化（70±6）
   - 第三个圆形：固定位置（250,350），半径变化（60±7）
3. 所有圆形均带有平滑的边缘过渡效果，颜色混合自然

## 核心渲染逻辑
`draw_circle` 方法的核心思路：
1. 遍历圆形外接矩形内的所有像素
2. 计算像素到圆心的距离，过滤超出半径的像素
3. 通过 `1 - (distance/radius)³` 计算平滑强度，实现边缘衰减
4. 混合前景色与背景色，生成过渡效果
5. 使用SDL2绘制单个像素点完成圆形渲染

## 操作说明
- 移动鼠标：第一个圆形会跟随鼠标指针位置移动
- 关闭窗口：点击窗口右上角关闭按钮即可退出程序

## 问题排查与解决方案
开发过程中遇到的关键问题及解决思路：

### 问题1：颜色通道错位（蓝显红、红显蓝）
- **现象**：设置纯蓝(0,0,255)显示为红色，纯红(255,0,0)显示为蓝色
- **根因分析**：SDL2的BGRA8888像素格式与代码中RGBA的颜色存储顺序不匹配，导致R/B通道解析颠倒
- **解决方案**：放弃手动管理帧缓冲（framebuffer）的像素格式，直接使用SDL2高层API（`SDL_SetRenderDrawColor`+`SDL_RenderDrawPoint`），由SDL自动适配底层像素格式，彻底解决通道错位问题

### 问题2：帧缓冲（framebuffer）绘制失效
- **现象**：尝试将像素写入framebuffer后更新到纹理，圆形无法显示
- **根因分析**：软渲染器经典架构（framebuffer→纹理→屏幕）的逻辑与SDL2高层绘制API冲突，手动管理的帧缓冲格式未匹配SDL纹理要求
- **解决方案**：简化架构，保留SDL2直接绘制的核心逻辑，移除无用的framebuffer代码，既保证功能正常，又精简代码结构

## 注意事项
1. 确保SDL2库已正确配置（头文件路径、链接库路径）
2. 程序运行时会占用少量CPU资源（因软渲染逐像素计算）
3. 窗口支持高DPI显示（SDL_WINDOW_ALLOW_HIGHDPI）
